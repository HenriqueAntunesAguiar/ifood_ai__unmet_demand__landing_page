Desenvolvendo a IA do Ifood!

Está IA do Ifood identifica pedidos de clientes dos quais o prato solicitado não tem na região, então a IA busca e informa aos restaurantes que podem fazer este prato, a recomendação de incluir no cardapio.

Relembrando que este é apenas um projeto para compartilhar um fluxo que PODE atender a demanda dita, não necessáriamente a demanda real da empresa em questão.

De onde veio esta IA?

Nosso Processo de Desenvolvimento

01

Criação dos Dados

Para poder realizar os testes, simulamos os dados, separando em base de dados Clientes e Resturantes.

02

Limpeza dos Dados

Após gerar os dados, é separado os dados por região para fazer a análise.

03

IA para Identificação Estilo do Restaurante

Criado um modelo para identificar se o novo prato condiz com o estilo do restaurante, por exemplo se a comida com demanda for sushi, o modelo identifica que não condiz com churrascarias premium e já tira do processo, economizando recursos.

04

IA para Coletar Dados do Novo Prato

Para este exemplo, foi criado uma IA que recebe a receita e retorna com os principais ingredientes para realizar, contudo poderia ser uma API também.

05

IA para Comparar os Ingredientes da Receita com do o Restaurante

Um terceiro modelo que avalia se os ingredientes presentes na nova receita condiz com os do restaurante e quais ingredientes o restaurante pode usar dos que já tem.

06

Relatório para os Restaurantes Selecionados

Integramos a solução de IA à plataforma Ifood e monitoramos continuamente seu desempenho.

Funcionalidades da AI

Analise de Perfil

Recomendação de pratos de acordo com o perfil do restaurante.

Informativo de Receitas

Elaboração de lista de ingredientes necessários para realizar a receita.

Comparação Inteligente de Viabilidade

Analise de viabilidade do novo prato para o restaurante com base nos ingredientes presentes no estabelecimento.

Fluxo do Processo

Fase 1: Base de Dados

Criação do banco de dados Cliente -- Localidade, Comida Pesquisada e Comida Encontratada na Localidade ou Não

Criação do banco de dados Restaurantes -- Localidade, Cardápio e Lista de Ingredientes

Fase 2: Tratamento dos Dados

Tratado e separado os dados por localidade, colocando os restaurantes e clientes por setor.

Assim, é possível determinar se o item pesquisado do cliente é correspondido pelos restaurantes na área.

Fase 3: Modelo de IA para Indentificar Perfil do Restaurante

O modelo vai comparar o novo prato que foi indentificado como demanda não atendida e analisar se tem relação com o tipo de restaurante e outros pratos do restaurante.

Esta etapa é importante para evitar gerar custo nos outros dois modelos de AI, pois se o perfil não for condizente, pode ser retirada da esteira de processos.

Fase 4: Modelo de IA para Informar os Principais Ingredientes da Nova Receita

Integração com a plataforma Ifood

Setembro 2023

Fase 5: Modelo de IA para Analisar Ingredientes da Receita e do Restaurante

Após todos os ingredientes da nova receita serem listados, é realizado uma comparação da receita com os ingredientes do restaurante por outro modelo de IA.

Assim, será enviada a lista de ingredientes da receita que o restaurante já tem, os que podem ser substituto na receita e os que faltam comprar.

Fase 6: Desenvolvimento do Relatório

Após todas estas etapas é finalizado um relatório informando restaurantes ápitos a receberem novos pratos, quantidade de possíveis novos clientes com as receitas, receita e a lista de análise da etapa anterior.

Este relatório deverá ser utilizado para construir uma mensagem para os restaurantes, repassando todas as informações.
